    Neste projeto, você pode ver a aplicação dos seguintes princípios: - SOLID
    - KISS
    - YAGNI
    - Testes unitários
    - Testes de integração
    - JavaDoc

    🏗 Fluxo de Atualização de Vagas

    👤 Cliente / Frontend
        |
        | HTTP Request (GET, POST, PUT, DELETE)
        v

    📝 Controller (JobController)
    - Recebe a requisição
    - Valida o input / DTO
    - Obtém companyId, path params e headers
    - Chama o caso de uso correspondente
        |
        v

    ⚙️ Use Case (CreateJobUseCase / UpdateJobUseCase / DeleteJobUseCase)
    - Implementa a lógica de negócio
    - Valida regras (ex: apenas a empresa dona pode editar a vaga)
    - Prepara a entidade
    - Chama o repositório
        |
        v

    💾 Repository (JobRepository)
    - Acessa o banco de dados
    - Cria, atualiza, deleta ou busca dados
        |
        v

    🗄 Banco de Dados
    - Persiste ou retorna entidades
        |
        v

    ⚙️ Use Case
    - Recebe o resultado do Repository
    - Aplica regras finais (se necessário)
        |
        v

    📝 Controller
    - Retorna o ResponseEntity para o cliente
